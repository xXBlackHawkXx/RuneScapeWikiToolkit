# IDE Agents

This folder is meant to be consumed by local IDE agents or coding assistants.

## Purpose

Give an agent enough architectural context to safely add, modify, or debug a module without reverse-engineering the whole app every time.

## Read order

1. `system-brief.md`
2. `architecture-map.md`
3. `module-blueprint.md`
4. `quality-checklist.md`
5. `prompts/new-module.prompt.md`
